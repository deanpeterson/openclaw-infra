# Adding a New Agent

## What You Write

To create a new agent, you provide **one file**: `AGENTS.md` — the agent's instructions, persona, and behavior. Everything else (metadata, ConfigMap, registration) is generated automatically.

Optionally, add a `JOB.md` for scheduled tasks.

```
agents/openclaw/agents/example-agent/
  AGENTS.md.envsubst       # ← You write this (agent instructions)
  JOB.md                   # ← You write this (optional, cron job)
  agent.json.envsubst      # ← Generated by add-agent.sh
  example-agent-agent.yaml.envsubst  # ← Generated by add-agent.sh
```

## Quick Start

### 1. Scaffold

```bash
./scripts/add-agent.sh --scaffold-only example-agent "Example Agent" "Monitors system health"
```

This creates the agent directory with all files. The script prompts for emoji and color.

### 2. Write your agent's instructions

Edit `AGENTS.md.envsubst` — this is the agent's identity. Everything between the YAML frontmatter and the end of the file becomes the agent's system prompt:

```markdown
---
name: ${OPENCLAW_PREFIX}_example_agent
description: Monitors system health
metadata:
  openclaw:
    emoji: "🔍"
    color: "#FF6B6B"
---

# Example Agent

You are Example Agent, a specialized AI agent.

## Your Role

Monitor system health and report anomalies.

## Tools

You have access to the `exec` tool for running bash commands (curl, cat, jq).

## Reporting

When you have findings to share, use `sessions_send` to message
${OPENCLAW_PREFIX}_${SHADOWMAN_CUSTOM_NAME}.
```

### 3. Deploy

```bash
./scripts/add-agent.sh example-agent
```

The script detects the existing files, builds the ConfigMap, registers the agent, and restarts the gateway. Your agent is ready to chat.

## Interactive Mode

Skip the scaffold step entirely — the script prompts for everything:

```bash
./scripts/add-agent.sh
```

You'll be prompted for ID, name, description, emoji, color, and whether to add a cron job. The script scaffolds and deploys in one step.

## AI-Assisted Workflow

Ask your AI assistant to create the `AGENTS.md.envsubst` file directly:

> "Create an agent called security-scanner that checks for CVEs in our container images using grype. It should report findings to the default agent."

Then deploy:

```bash
./scripts/add-agent.sh security-scanner
```

The script auto-generates `agent.json` and the ConfigMap from your AGENTS.md.

## Flags

| Flag | Purpose |
|------|---------|
| `--k8s` | Use kubectl instead of oc |
| `--scaffold-only` | Create files only, don't deploy |
| `--env-file PATH` | Custom .env file |

## Adding a Scheduled Job

Create a `JOB.md` in your agent's directory:

```bash
cp agents/openclaw/agents/_template/JOB.md.template \
   agents/openclaw/agents/example-agent/JOB.md
```

Edit the frontmatter:

```yaml
---
id: example-agent-job        # Unique job ID
schedule: "0 */8 * * *"      # Every 8 hours
tz: UTC                      # Timezone
---
```

Write the job instructions in the body — this is the message your agent receives
when the job fires. Then update the running jobs:

```bash
./scripts/update-jobs.sh           # OpenShift
./scripts/update-jobs.sh --k8s     # Kubernetes
```

### Cron Schedule Examples

| Schedule | Expression |
|----------|-----------|
| Every day at 9 AM UTC | `0 9 * * *` |
| Every 8 hours | `0 */8 * * *` |
| Weekdays at 9 AM and 5 PM | `0 9,17 * * 1-5` |
| Every Monday at 8 AM | `0 8 * * 1` |

## Template Variables

These envsubst variables are substituted at deploy time:

| Variable | Purpose |
|----------|---------|
| `${OPENCLAW_PREFIX}` | Your namespace prefix (e.g., `sally`) |
| `${OPENCLAW_NAMESPACE}` | Full namespace (e.g., `sally-openclaw`) |
| `${SHADOWMAN_CUSTOM_NAME}` | Default agent's ID suffix (for `sessions_send`) |
